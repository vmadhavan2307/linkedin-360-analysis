## 📋 Quick File Guide

### `README.md` ⭐⭐⭐ (Most Important)
**What:** Main landing page for your repo  
**Who reads it:** Everyone who discovers your repo  
**Content:** Overview, three frameworks, quick start, FAQ, how to share  
**Key section:** "Quick Start (2 minutes)"  

### `linkedin-360-skill.md` ⭐⭐⭐ (Core Skill)
**What:** The actual Claude skill people copy  
**Who reads it:** Users who want to analyze their recommendations  
**Content:** Framework definitions, workflow, edge cases, output specs  
**Key section:** "Framework Selection" and individual framework definitions  
**How to share:** Copy the raw version for easy copy-paste  

### `USAGE-GUIDE.md` ⭐⭐ (How-To)
**What:** Step-by-step instructions for using the skill  
**Who reads it:** Users who want clear walkthroughs  
**Content:** Prerequisites, step-by-step guide, tips, troubleshooting  
**Key section:** "Step 1–7" workflow  

### `example-output.html` ⭐⭐ (Showcase)
**What:** Your full 360-degree analysis across all three frameworks  
**Who views it:** People wanting to see what output looks like  
**Content:** Universal, Executive, and Reforge PM analyses  
**Key section:** Interactive tabs showing all three frameworks  
**Why it matters:** Builds confidence that the skill actually works  

### `landing-page.html` ⭐ (Alternative Share)
**What:** Standalone webpage explaining the skill  
**Who visits it:** People you send the link to  
**Content:** Features, quick start, frameworks, FAQ, CTA  
**Use case:** Share as alternative to GitHub repo  

### `REDDIT-POST-TEMPLATE.md` ⭐ (Marketing)
**What:** 5 copy-paste Reddit post templates  
**Who uses it:** You, to share on Reddit  
**Content:** Different angles for different subreddits  
**Templates:** r/careerguidance (×2), r/productivity, r/jobs, r/ProductManagement  
**Pro tip:** Customize slightly; don't post identical messages  

### `GITHUB-SETUP-GUIDE.md` ⭐ (Setup)
**What:** Instructions to create and manage the GitHub repo  
**Who reads it:** You, when setting up  
**Content:** Quick setup (5 min) and proper setup (15 min) options  
**Key sections:** Step-by-step GitHub creation, troubleshooting  

### `CONTRIBUTING.md` ⭐ (Community)
**What:** Guidelines for people who want to contribute  
**Who reads it:** Contributors, community members  
**Content:** Ways to contribute, PR process, code of conduct  
**Key section:** "Ways to Contribute" and "Pull Request Process"  

### `LICENSE` ⭐ (Legal)
**What:** MIT open-source license  
**Who cares:** Anyone using your code, legal clarity  
**Meaning:** People can use, modify, share (with attribution)  

### `.gitignore` ⭐ (Technical)
**What:** Git configuration (what to ignore)  
**Who needs it:** Only if you use Git locally  
**Purpose:** Prevents unnecessary files from being tracked
